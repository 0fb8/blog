# I'm Launching This Blog

I'm developing this personal blog site by myself. 
This site is built with a Static Site Generator - SSG.

## What is an SSG?

Basically, static website like a personal blog ultimately only requires HTML files. 
The easist and most primitive way is to manually write an HTML file for every article. 
However it is troublesome for humans because HTML is designed for browsers, not for people. 
When writing an article, we prefer to use a human-friendly language like Markdown, which is then automatically converted into HTML files. 
This concept is what we call an SSG. 

In reality, the source language written by human doesn't have to be Markdown. Any language is essentially a tree (cf: Abstract Syntax Tree), and HTML is also essentially a tree structure usually called the Document Object Model. 
What matters is "the definition of how to convert one tree into another", and only it. 
As long as you develop a parser, you can even use your own custom syntax. 

## Why build this from scratch?

There are many great SSG frameworks such as Hugo, Astro, Hono, and so on.
I initially considered using them, but I realized I would have to learn their specific usages. 
If so, I thought making my own SSG from zero would be better in perspectives of both customizability and gaining a deeper understanding of the fundamentals. 

## Why English?

My native language is Japanese, but I've chosen to write this in English for two reasons.

1. **For Learning:** I struggle with English output. This blog is a precious opportunity to express my thoughts in a foreign language.

2. **For Satisfaction:** If I wrote this in Japanese, I would think "how thin the content is!" and wouldn't feel to post it. Whereas writing in English make me spend a lot of time and energy, so the hurdle for the content itself feels lower. 

After writing a draft on my own, I ask an AI to check and advise my English. So while this article consists my own touch, it would have few crucial mistakes of English. 
Considering the current AI is accurately called a Large **Language** Model, using it for **language** learning is, in a sense, its most straightforward use case, isn't it?
